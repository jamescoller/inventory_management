# PR-B — MQTT Telemetry Mirror / Consumer (Phase 16.1, part 2 of 2)

**Date:** 2026-06-10
**Status:** approved (design); pending implementation
**Depends on:** PR-A (WAL + DB-directory migration) — **shipped** (PR #142). DB is WAL at
`~/inventory_db_dir/inventory_db.sqlite3`, directory-mounted at `/app/db`; `SQLITE_DB_PATH`
is set in the compose `environment:` (not `~/.env_inventory`, which is root-owned).
**Parent design:** [`docs/bambu-mqtt-integration.md`](../../bambu-mqtt-integration.md) §1–2, §4
**Roadmap:** `todo.md` Phase 16.1 (PR-B)

## 1. Goal & boundary

A **read-only telemetry mirror.** A new container subscribes to each Bambu printer's local
MQTT broker and upserts the latest state (plus a downsampled time-series) into **its own
tables only.** It is **fully decoupled from `InventoryItem`**: no spool/RFID matching, no
auto-`PrintJob`, no HMS→`MaintenanceEvent` writes. Those are **Phase 16.3**. We *capture* the
16.3 join keys now (`AMSChannelState.tray_uuid`, `PrinterState.task_id`,
`PrinterState.hms_codes`) so 16.3 needs no schema change.

**Verified against real hardware (2026-06-10):** a live `pushall` probe of H2Laser
(`10.10.30.11`) returned 357 fields under a single top-level `print` object. The field
selection below is mapped from that real payload. 2 of 4 printers were reachable (the other 2
were powered off) — confirming printers are **intermittently reachable** and the consumer must
tolerate per-printer disconnects.

## 2. Data model (migration `0034`, in `inventory/models.py`)

Bambu sends many numerics **as strings** (`humidity:'5'`, `temp:'24.6'`, `nozzle_temper:23.0`
is a float, `mc_percent:100` is an int). The ingest layer (§3) coerces; models store typed
values. All timestamps via `auto_now`.

### `PrinterDevice` — registry / connection config
| Field | Type | Notes |
|---|---|---|
| `serial` | `CharField(max_length=100, unique=True)` | matches `InventoryItem.serial_number` width |
| `name` | `CharField(max_length=100)` | "Scooby Doo" etc. |
| `ip_address` | `GenericIPAddressField` | |
| `access_code` | `CharField(max_length=32, blank=True)` | LAN pairing code (stored per James's decision) |
| `model_name` | `CharField(max_length=50, blank=True)` | "H2D", "X1C"… |
| `item` | `FK(InventoryItem, null, blank, SET_NULL, related_name="printer_device")` | optional curated machine link (16.3); never guessed |
| `enabled` | `BooleanField(default=True)` | consumer only dials enabled rows |
| `last_seen_at` | `DateTimeField(null, blank)` | updated on each message; observability |

### `PrinterState` — latest snapshot, `OneToOne(PrinterDevice, CASCADE)`, upserted
`gcode_state` (Char, blank), `mc_percent` (PositiveSmallInt, null), `layer_num` /
`total_layers` (PositiveInt, null), `nozzle_temp` / `nozzle_target` / `bed_temp` /
`bed_target` (Decimal(5,1), null), `remaining_min` (PositiveInt, null), `subtask_name` (Char,
blank), `task_id` (Char, blank — `print.task_id`/`subtask_id`), `hms_codes` (JSONField,
default=list — list of `{code, attr}`), `updated_at`.
*Source:* `print.gcode_state/mc_percent/layer_num/total_layer_num/nozzle_temper/
nozzle_target_temper/bed_temper/bed_target_temper/mc_remaining_time/subtask_name/task_id/hms`.

### `AMSUnitState` — per AMS unit, `unique_together(device, ams_index)`, upserted **(NEW)**
Captures that **AMS 2 Pro / AMS HT double as dryers.** `device` (FK→PrinterDevice, CASCADE),
`ams_index` (PositiveSmallInt), `humidity` (PositiveSmallInt, null — the 1–5 level),
`humidity_raw` (PositiveSmallInt, null), `temp` (Decimal(4,1), null — °C), `dry_time`
(PositiveInt, null — minutes), `dry_duration` / `dry_temperature` (Int, null — active
`dry_setting`), `dry_filament` (Char, blank), `updated_at`.
*Source:* `print.ams.ams[i].{id,humidity,humidity_raw,temp,dry_time,dry_setting.*}`.
*Why its own table:* humidity/temp/dry_* live at the **unit** level (`ams[i]`), not the tray
level — normalizing avoids repeating them on all 4 tray rows. Enables a future "has this
filament been dried?" workflow (16.3, a write to inventory).

### `AMSChannelState` — per tray, `unique_together(device, ams_index, tray_index)`, upserted
`device` (FK CASCADE), `ams_index`, `tray_index` (PositiveSmallInt), `tray_uuid` (Char, blank —
**RFID join key for 16.3**), `tray_info_idx` (Char, blank), `tray_type` (Char, blank),
`tray_sub_brands` (Char, blank), `color_hex` (Char(8), blank — `RRGGBBAA`), `remain_pct`
(SmallInt, null — −1 = unknown, stored as-is), `updated_at`.
*Source:* `print.ams.ams[i].tray[j].{id,tray_uuid,tray_info_idx,tray_type,tray_sub_brands,
tray_color,remain}`.

### `TelemetrySample` — append-only trend store
`device` (FK CASCADE), `ts` (DateTimeField), `gcode_state` (Char, blank), `mc_percent`
(PositiveSmallInt, null), `nozzle_temp` / `bed_temp` (Decimal(5,1), null), `remaining_min`
(PositiveInt, null). `Meta: indexes=[Index(fields=["device", "ts"])]`. **Downsampled** (§3.3) —
printer print-state only. (AMS-humidity *trends*, if ever wanted, are an HA/16.2 concern;
`AMSUnitState` holds the latest, which satisfies the "is it dry now?" use-case.)

**Dropped from scope** (reviewed against the live payload): external spool `vir_slot`
(echoed manual data, untrusted), `wifi_signal` (HA's job), all camera/firmware/laser-`2D`/
xcam/fan/cloud/network fields. **Deferred:** nozzle wear/type (`device.nozzle.info`) → the
maintenance phase (feeds `NozzleConfig`/inventory), not the mirror.

## 3. Consumer — logic/transport split (mirrors `audit.py` + views)

### 3.1 `inventory/telemetry.py` — pure, unit-testable core
- `ingest_report(device, payload: dict) -> None` — takes the parsed `print` object and
  **delta-merges**: updates only the **keys present** in this message. Bambu sends a full
  snapshot on `pushall`, then **partial deltas** — so we must never null-out a field the delta
  omitted. Upserts `PrinterState` (+`get_or_create`), each present `ams[i]` →`AMSUnitState`,
  each present `ams[i].tray[j]` →`AMSChannelState`; appends a `TelemetrySample` per §3.3.
  - `hms` is replace-when-present (it arrives as the full active list, not a per-code delta);
    absent → leave unchanged.
  - Coercion helpers tolerate str/float/int/None/missing: `_to_int`, `_to_decimal`,
    `_to_pct` (keeps −1).
- `should_sample(prev: TelemetrySample | None, new_state: str, *, now, interval_s=300) -> bool`
  — True if `prev is None`, `new_state != prev.gcode_state`, or
  (`new_state == "RUNNING"` and `now - prev.ts >= interval_s`). `now` is a param for
  deterministic tests.

### 3.2 `management/commands/run_telemetry_consumer.py` — MQTT transport
- **Startup:** retry-until-ready loop — `PrinterDevice.objects.filter(enabled=True)` catching
  `OperationalError` (schema not migrated yet; the `telemetry` service may start before `web`
  finishes `migrate`), sleeping with backoff up to ~60s.
- **Per enabled device:** one `paho.mqtt.Client(CallbackAPIVersion.VERSION2)`, `username_pw_set
  ("bblp", device.access_code)`, `tls_set(cert_reqs=CERT_NONE)` + `tls_insecure_set(True)`
  (self-signed), `reconnect_delay_set(min_delay=1, max_delay=120)`, `loop_start()` (own thread,
  auto-reconnect). `on_connect` → subscribe `device/{serial}/report` + publish `pushall` to
  `device/{serial}/request`. `on_message` → `json.loads` → guard `"print" in payload` →
  `close_old_connections()` → `ingest_report(device, payload["print"])` → bump `last_seen_at`,
  all wrapped in try/except+log so one bad message never kills the thread.
- **Offline printers** (you saw 2/4): connection just retries with backoff; never blocks other
  printers or crashes the command. `on_disconnect` logs.
- **SQLite write concurrency:** N printer threads each get a thread-local Django connection;
  WAL + `busy_timeout=5000` (from PR-A) serialize the low-volume writes. The single-writer
  `queue.Queue` fallback (design doc §3.4) stays **unbuilt** unless contention is *measured*.

### 3.3 Downsampling
`ingest_report` always upserts the latest-state tables (cheap single-row writes). It appends a
`TelemetrySample` **only** when `should_sample()` is true — i.e. on `gcode_state` transition or
every `interval_s` (300s) while `RUNNING`. The command tracks the last sample per device
**in-memory** (lost on restart → first post-restart message just creates a fresh sample; the
DB is the source of truth via the latest row if needed).

## 4. Compose service, seeding, admin

- **`docker-compose.yml` — 3rd service `telemetry`:** `build: .`, `command: ["python",
  "manage.py", "run_telemetry_consumer"]`, `restart: unless-stopped`, mounts
  `${HOME}/inventory_db_dir:/app/db`, `environment: SQLITE_DB_PATH=/app/db/inventory_db.sqlite3`
  + `env_file: ${HOME}/.env_inventory`, `depends_on: [web]`. **Does NOT run migrations** (web's
  `entrypoint.sh` owns that; only one migrator). `deploy.sh` needs no change (no DB-path change).
- **`management/commands/seed_printer_devices.py`:** idempotent `get_or_create(serial=…)` for
  the 4 printers (serial/name/ip/model from `docs/bambu-mqtt-integration.md` §6). Access codes
  **not** hardcoded in committed code — James fills them via admin or an optional
  `--codes serial=code,…` arg.
- **`admin.py`:** `PrinterDeviceAdmin` (editable `access_code`/`enabled`/`item`); read-only
  `list_display` admins for `PrinterState`, `AMSUnitState`, `AMSChannelState`, `TelemetrySample`
  to eyeball the mirror before 16.2 dashboards exist.

## 5. Testing (`manage.py test`; no live MQTT)

- **Fixture:** a trimmed copy of **tonight's real H2Laser payload** committed at
  `inventory/tests/fixtures/bambu_report_h2laser.json` (the `print` object: state/temps/progress
  + one `ams[0]` with 2 trays + `hms`; camera/laser/cloud noise stripped).
- `ingest_report` from the full fixture → asserts `PrinterState`, `AMSUnitState` (humidity/temp/
  dry_*), `AMSChannelState` (uuid/type/color/`remain`) rows match.
- **Delta no-clobber:** ingest full snapshot, then a partial delta (`{"mc_percent": 42}`) →
  assert `gcode_state`/temps unchanged, `mc_percent` updated.
- **`remain: -1`** stored as −1 (unknown), not coerced to a positive.
- **`should_sample`:** None→True; same-state-before-interval→False; state-change→True;
  RUNNING past interval→True (explicit `now`).
- **`seed_printer_devices`:** creates 4 rows; re-run creates 0 (idempotent).
